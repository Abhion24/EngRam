1. Executive Summary 
Engram is a VS Code extension that gives Claude Code a persistent, long-term memory of your 
codebase. Every time a developer opens a new Claude Code session, they waste 10–20 minutes re-
explaining their project — the stack, the architecture, recent decisions, known bugs. Engram eliminates 
this entirely. 
 
The extension silently indexes your codebase on first activation, records high-signal decisions from 
every Claude Code session, and automatically injects the right context at the start of every new 
session. Claude starts informed. No re-explaining. No lost context. No repeated bugs. 
 
Engram does not wrap Claude Code. It fixes the fundamental infrastructure gap in how Claude 
Code operates today — the absence of memory across sessions. 
 
1.1 The One-Line Pitch 
Claude Code is amnesiac. Engram is the brain it was never given.

3. Solution — How Engram Works 
Engram introduces a three-layer memory architecture that runs silently alongside Claude Code. It is 
fully automatic — developers do nothing differently. Engram works in the background. 
 
3.1 The Three Memory Layers 
 
01 
Codebase Indexer 
On first activation, Engram deeply scans your entire project. It uses Tree-sitter to parse 
every file — extracting components, functions, routes, DB schema, dependencies, and 
architectural patterns. All extracted data is converted into semantic vector embeddings and 
stored in a local ChromaDB instance on your machine. Nothing leaves your computer. 
 
02 
Session Memory Recorder 
As you work with Claude Code, Engram listens. It watches file diffs after each session to 
detect what changed and infers why. It extracts high-signal memories — bugs fixed, 
decisions made, patterns established, approaches avoided — and stores them as structured 
entries tagged by type, file, and timestamp. 
 
03 
Context Injector 
When a new Claude Code session opens, Engram performs a semantic similarity search 
across all stored memories. It retrieves the most relevant entries for your current task, 
compresses them into a compact context block, and injects it into Claude's system prompt 
automatically. Claude starts the session already knowing your project. 
 
04 
Memory Dashboard 
A local web dashboard at localhost:6173 gives you full visibility and control. See all 
memories on a timeline, search by keyword or file, pin critical memories that should always 
be injected, edit incorrect entries, and export your entire project memory as a portable 
.engram file. 
 
05 
Continuous Learning 
Every session makes Engram smarter. The memory grows richer over time. The longer you 
use it, the more precisely Claude understands your project — your patterns, your 
preferences, your pitfalls, and your standards.

used for similarity search in ChromaDB 
ChromaDB 
An open-source vector database for storing and querying embedding-
based memories locally 
Tree-sitter 
A language-agnostic code parsing library that produces Abstract 
Syntax Trees — used for codebase indexing 
Context Injection 
The act of automatically prepending relevant memories to Claude's 
system prompt at session start 
Session 
A single Claude Code conversation — Engram records memories from 
each session as it ends 
Semantic Search 
Finding memories based on meaning similarity rather than exact 
keyword match 
Token Budget 
The maximum number of tokens available in Claude's context window 
— Engram manages this carefully 
.engram File 
A portable JSON export of a project's full memory store — can be 
shared between developers
